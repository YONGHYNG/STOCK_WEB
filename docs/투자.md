# BTCUSDT 선물 자동매매 전략 기준

## 현재 구현 기준

전략 엔진은 거래량, 추세, RSI만 핵심 기준으로 사용한다. 실주문은 기본 비활성이고, 주문 인터페이스는 paper trading을 기본값으로 둔다.

주요 구현 파일:

- `backend/market/candle_collector.py`
- `backend/strategy/indicator.py`
- `backend/strategy/strategy.py`
- `backend/strategy/volume_trend_engine.py`
- `backend/risk/risk_manager.py`
- `backend/order/order_manager.py`
- `backend/strategy/main_loop.py`

## 캔들

`CandleCollector`는 1분봉과 5분봉을 가져올 수 있고, 실제 Bitget API 호출부와 테스트용 mock provider를 분리한다.

모든 전략 판단은 확정 캔들 기준으로 수행한다.

## 지표

pandas 기반으로 다음 지표를 계산한다.

- `MA90`
- `MA200`
- `RSI14`
- `ATR14`
- `volume_ma20`
- `volume_ratio`

## 전략 신호

### A. 반등 숏

신호: `SHORT_REBOUND`

- 하락 추세: `close < MA90 < MA200`
- 가격이 MA90 또는 MA200 근처까지 반등
- 고가는 MA를 터치했지만 종가는 MA 아래
- RSI가 55 이상에서 하락 전환
- `volume_ratio >= 1.0`

### B. 저점 이탈 후 되돌림 숏

대기 신호: `WAIT_RETEST_SHORT`

- 최근 20개 캔들의 최저가를 `support_level`로 계산
- `close < support_level`
- `volume_ratio >= 1.2`

진입 신호: `SHORT_BREAKDOWN`

- 이후 고가가 `support_level`을 터치
- 종가는 `support_level` 아래
- `RSI < 50`

### C. 저점 지지 후 롱

신호: `LONG_BOUNCE`

- `low <= support_level and close > support_level`
- 아래꼬리가 윗꼬리보다 김
- RSI가 40 이하에서 상승 전환
- `volume_ratio >= 1.0`

### D. 추세 전환 후 눌림 롱

대기 신호: `WAIT_PULLBACK_LONG`

- 이전 close는 MA90 아래
- 현재 close는 MA90과 MA200 위
- `RSI > 50`
- `volume_ratio >= 1.2`

진입 신호: `LONG_TREND_CHANGE`

- 이후 `low <= breakout_level and close > breakout_level`
- RSI가 45~60

## 리스크 관리

전략 전용 `StrategyRiskManager` 기준:

- 1회 거래 손실은 계좌의 0.5% 이하
- 하루 손실 -2% 도달 시 신규 진입 금지
- 연속 손실 3회면 자동매매 정지
- API 오류 3회면 자동매매 정지
- 손절 후 같은 방향 재진입은 최소 3개 캔들 대기

## 주문

`OrderManager`는 다음 인터페이스를 제공한다.

- `place_long`
- `place_short`
- `close_position`

기본값은 `paper_trading=True`이며, 실거래 전에는 실제 Bitget 주문을 보내지 않는다.

## 루프

`MainLoop`는 다음 순서로 동작한다.

1. 1분마다 확정 캔들 조회
2. 지표 계산
3. 전략 신호 판단
4. 리스크 매니저로 진입 가능 여부 확인
5. 주문 매니저로 paper 주문 실행
6. 모든 신호와 주문 결과 로그 저장

거래량 과열 돌파 직후에는 시장가 추격 진입을 막기 위해 `WAIT_RETEST_SHORT`, `WAIT_PULLBACK_LONG` 상태를 먼저 사용한다.
