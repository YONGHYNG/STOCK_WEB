# 선물 단타 자동매매 판단 기준

## 현재 구현 기준 요약

현재 시스템은 실제 주문 실행이 아니라 `LONG`, `SHORT`, `HOLD` 방향과 진입 등급, 리스크 경고, 진입 사유를 반환하는 분석 엔진이다.

주요 구현 파일:

- `backend/strategy/scalping_signal_engine.py`
- `backend/strategy/multi_timeframe_strategy.py`

기존 호환을 위해 `long_probability`, `short_probability` 필드는 유지하지만, 의미는 승률이 아니라 `LONG 점수`, `SHORT 점수`다.

## 입력 데이터

- `5m`, `15m`, `30m`, `1H`, `6H`, `1D` OHLCV 캔들
- 현재가 또는 최신 종가
- 스프레드
- 펀딩비
- 선택값: 레버리지, 예상 청산가, 호가 정보, 수수료율

## 계산 지표

각 시간봉마다 다음 지표를 계산한다.

- `EMA20`, `EMA60`
- `RSI14`
- `MACD 12,26,9`, `MACD histogram`
- `ATR14`
- `volume_sma20`, `volume_ratio`
- `upper_wick_ratio`, `lower_wick_ratio`, `body_ratio`
- `recent_high`, `recent_low`
- `failed_breakout`, `failed_breakdown`

## 시장 모드

점수 계산 전에 시장 모드를 먼저 분류한다.

- `TREND_LONG`
- `TREND_SHORT`
- `REVERSAL_SHORT`
- `REVERSAL_LONG`
- `RANGE`
- `HOLD`

강한 상승 자체는 `SHORT` 신호가 아니며, 강한 하락 자체도 `LONG` 신호가 아니다. 반대 방향 진입은 실패 신호가 충분할 때만 허용한다.

## 시간봉 역할

- `5m`: 진입 타이밍
- `15m`: 단기 방향 확인
- `30m`: 단기 추세 확인
- `1H`: 방향 신뢰도 확인
- `6H`: 리스크 필터
- `1D`: 큰 시장 환경 리스크 필터

`6H`, `1D`는 방향을 죽이는 기준이 아니라 등급 제한, 수량 축소, 경고 추가 용도다.

## 점수 기준

분석 시작 점수는 항상 다음과 같다.

```python
long_score = 50
short_score = 50
direction = "HOLD"
market_mode = "NONE"
entry_grade = "F"
warnings = []
reasons = []
```

`TREND_LONG`에서는 EMA 정렬, 현재가의 5m EMA20 상단 위치, MACD histogram 증가, 적정 RSI, 거래량 증가가 `LONG` 점수에 반영된다.

`TREND_SHORT`에서는 EMA 하락 정렬, 현재가의 5m EMA20 하단 위치, MACD histogram 감소, 적정 RSI, 거래량 증가가 `SHORT` 점수에 반영된다.

RSI 과열은 바로 숏 신호가 아니고, RSI 과매도는 바로 롱 신호가 아니다. 과열/과매도는 추격 진입 경고와 해당 추세 방향 감점으로만 처리한다.

## 역추세 기준

`REVERSAL_SHORT`는 상승 과열 후 실패 조건이 4개 이상일 때만 허용한다.

- 5m RSI 80 이상
- 긴 윗꼬리
- 거래량 급증
- 신고가 돌파 실패
- MACD histogram 감소
- 5m 종가 EMA20 이탈

`REVERSAL_LONG`은 하락 과열 후 회복 조건이 4개 이상일 때만 허용한다.

- 5m RSI 20 이하
- 긴 아랫꼬리
- 거래량 급증
- 신저가 이탈 후 회복
- MACD histogram 하락폭 둔화
- 5m 종가 EMA20 회복

역추세 매매는 최대 `B`등급이며, `position_size_ratio`는 0.5 이하로 제한한다.

## 리스크 필터

다음 조건은 진입 금지 또는 등급 제한으로 처리한다.

- 스프레드 과도
- ATR 변동성 과도
- 손익비 부족
- 청산가 안전거리 부족
- 캔들 또는 거래량 데이터 부족
- 5m/15m/30m 주요 방향 충돌
- LONG/SHORT 점수가 동시에 높은 충돌 상태

펀딩비는 방향 신호가 아니라 포지션 쏠림 경고로만 사용한다.

## 손절과 익절

손절과 익절은 5m ATR 기준으로 계산한다.

LONG:

- 손절가 = `min(직전 스윙 저점, entry_price - ATR * 1.2)`
- 1차 익절 = `entry_price + risk * 1.0`
- 2차 익절 = 추세 `risk * 1.5`, 역추세 `risk * 1.2`

SHORT:

- 손절가 = `max(직전 스윙 고점, entry_price + ATR * 1.2)`
- 1차 익절 = `entry_price - risk * 1.0`
- 2차 익절 = 추세 `risk * 1.5`, 역추세 `risk * 1.2`

## 최종 방향

최종 방향은 점수, 시장 모드, 리스크 필터를 모두 통과해야 결정된다.

- `LONG`: `long_score >= 65`, `short_score <= 40`, 시장 모드가 `TREND_LONG` 또는 `REVERSAL_LONG`
- `SHORT`: `short_score >= 65`, `long_score <= 40`, 시장 모드가 `TREND_SHORT` 또는 `REVERSAL_SHORT`
- 그 외는 `HOLD`

애매하면 무조건 `HOLD`한다. 진입 등급이 `C`, `D`, `F`이면 최종 방향도 `HOLD`다.
