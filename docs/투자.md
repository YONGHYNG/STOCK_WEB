# 투자 진입가 산정 기준

## 현재 구현 기준 요약

현재 시스템은 Bitget BTCUSDT USDT-M 무기한 선물 기준으로 가격을 분리해서 판단한다.

- 분석 기준가: 5분봉 등 선택 시간봉의 최신 캔들 종가
- 현재 체결가: Bitget ticker의 Last Price
- 위험 기준가: Mark Price
- 예상 LONG 진입가: 매도 1호가에 슬리피지 버퍼 반영
- 예상 SHORT 진입가: 매수 1호가에 슬리피지 버퍼 반영
- 손절/익절: 예상 진입가 기준 ATR 1.5배 손절, 1.5R/2.5R/3.5R 익절
- 방향 판단: LONG 점수와 SHORT 점수를 별도로 계산
- 진입 등급: A/B만 자동매매 후보, C/D/F는 관망 또는 차단
- 포지션 크기: 계좌 equity의 1% 손실 한도와 손절 거리로 BTC 수량 계산

주요 구현 파일:

- `backend/futures_ai_engine.py`
- `backend/bitget_client.py`
- `backend/risk_manager.py`
- `backend/backtester.py`

기존 `long_probability`, `short_probability` 필드는 API/DB/프론트 호환을 위해 유지하지만, 의미는 실제 승률이 아니라 `LONG 점수`, `SHORT 점수`이다.

---

이 문서는 현재 코드 기준으로 AI가 투자 방향과 진입가를 정하는 이유를 설명한다.

## 진입가 기준

현재 진입가는 예측 호가나 별도 주문가가 아니라, 분석 기준 시간봉의 최신 종가다.

기본 분석 시간봉은 `5m`이며, 코드에서는 마지막 캔들의 `close` 값을 사용한다.

```python
entry = float(last["close"])
```

즉 화면에 표시되는 `진입가`는 현재 5분봉 기준으로 가장 최근에 확정 또는 수신된 종가다.

## 방향 판단 기준

AI는 `LONG` 점수를 50점에서 시작한 뒤 EMA, MACD, RSI, 거래량 조건으로 점수를 더하거나 뺀다.

최종적으로:

- `LONG 확률 >= 60`이면 `LONG`
- `SHORT 확률 >= 60`이면 `SHORT`
- 둘 다 아니면 `HOLD`

`SHORT 확률`은 `100 - LONG 확률`로 계산된다.

## EMA 기준

EMA는 추세 판단에 사용한다.

- `EMA20 > EMA60`: 중기 추세가 우상향이라고 보고 LONG 점수 `+12`
- `EMA20 < EMA60`: 중기 추세가 약하다고 보고 LONG 점수 `-12`

의미:

- EMA20은 최근 가격 흐름에 더 민감하다.
- EMA60은 더 긴 흐름을 본다.
- EMA20이 EMA60 위에 있으면 최근 가격 흐름이 중기 흐름보다 강하다고 판단한다.
- 반대로 EMA20이 EMA60 아래에 있으면 하락 또는 약세 흐름으로 본다.

## RSI 기준

RSI는 과매수, 과매도, 중립 구간 판단에 사용한다.

- `35 <= RSI <= 65`: 과열이 아닌 중립 구간으로 보고 LONG 점수 `+6`
- `RSI < 20`: 과매도지만 투매 위험도 있다고 보고 LONG 점수 `+3`
- `RSI < 30`: 과매도 구간에 가까워 반등 가능성이 있다고 보고 LONG 점수 `+8`
- `RSI > 70`: 과열 구간으로 보고 LONG 점수 `-8`
- `RSI >= 80`: 추격매수 위험이 크다고 보고 LONG 점수 `-15`

의미:

- RSI가 너무 높으면 이미 많이 오른 상태로 보고 LONG 진입 위험을 낮춘다.
- RSI가 30 아래면 반등 가능성을 반영한다.
- RSI가 20 아래로 너무 낮으면 단순 반등보다 투매 위험도 같이 보기 때문에 가산점을 낮게 준다.
- RSI가 35~65면 과열이나 투매가 아닌 안정적인 구간으로 본다.

## MACD 기준

MACD 히스토그램은 모멘텀 판단에 사용한다.

- MACD 히스토그램이 `0보다 크고`, 이전보다 증가하면 LONG 점수 `+10`
- MACD 히스토그램이 `0보다 작고`, 이전보다 감소하면 LONG 점수 `-10`

의미:

- 양수이면서 증가 중이면 상승 모멘텀이 커지는 것으로 본다.
- 음수이면서 더 낮아지면 하락 모멘텀이 강해지는 것으로 본다.

## 거래량 기준

거래량은 현재 추세의 신뢰도를 보정하는 데 사용한다.

조건:

- 현재 거래량이 `20봉 평균 거래량 * 1.5` 이상이면 거래량 급증으로 본다.

보정:

- 거래량 급증 + `EMA20 > EMA60`: 상승 추세가 거래량을 동반한다고 보고 LONG 점수 `+6`
- 거래량 급증 + `EMA20 < EMA60`: 하락 추세에서 변동성이 커졌다고 보고 LONG 점수 `-6`

## 멀티 타임프레임 보정

기본 판단은 5분봉을 중심으로 하지만, 여러 시간봉 방향을 함께 반영한다.

각 시간봉이 `LONG`이면 LONG 점수에 가산하고, `SHORT`이면 감산한다.

가중치:

- `5m`: 2
- `15m`: 3
- `30m`: 4
- `1H`: 5
- `6H`: 6
- `1D`: 7
- `1W`: 5
- `1M`: 3

의미:

- 짧은 시간봉보다 `1H`, `6H`, `1D` 같은 상위 시간봉을 더 중요하게 본다.
- 5분봉이 LONG이어도 상위 시간봉이 SHORT이면 최종 방향이 SHORT이나 HOLD로 바뀔 수 있다.

## 손절가와 익절가 기준

손절가와 익절가는 ATR 기준으로 계산한다.

ATR은 최근 변동폭을 나타내는 지표다.

현재 설정:

- `ATR_STOP_MULTIPLIER = 1.5`
- `TAKE_PROFIT_R_MULTIPLIER = 2.0`

LONG일 때:

- 손절가 = 진입가 - `ATR14 * 1.5`
- 1차 익절가 = 진입가 + 위험폭 * `2.0`
- 2차 익절가 = 진입가 + 위험폭 * `3.0`

SHORT일 때:

- 손절가 = 진입가 + `ATR14 * 1.5`
- 1차 익절가 = 진입가 - 위험폭 * `2.0`
- 2차 익절가 = 진입가 - 위험폭 * `3.0`

## Confidence 의미

현재 `confidence`는 투자 성공 확률이 아니다.

계산식:

```python
confidence = abs(long_probability - short_probability)
```

의미:

- LONG과 SHORT 판단이 얼마나 한쪽으로 기울었는지를 나타낸다.
- 예를 들어 LONG 70%, SHORT 30%이면 confidence는 40%다.
- 따라서 confidence는 승률이 아니라 방향 신뢰도에 가깝다.

## 요약

현재 진입가는 최신 5분봉 종가다.

방향은 다음 조건을 종합해 결정한다.

- EMA20과 EMA60으로 추세 판단
- RSI로 과열/과매도 판단
- MACD 히스토그램으로 모멘텀 판단
- 거래량 급증 여부로 추세 신뢰도 보정
- 5m부터 1M까지 멀티 타임프레임 방향성 보정

손절과 익절은 고정 퍼센트가 아니라 ATR 변동성 기준으로 계산한다.
